- when either the query or hit collection is not provided, the query or hit object that will be registered is a custom object - import.BLAST: several XML tags can be combined to generate either a query or hit id. Python code can also be used to modify the content on those tags on the fly: Example 1: ... 1 1 CH0704v-contig00010 length=3963 numreads=678 3963 ... + --query-id-getter ".split()[0]" = 'CH0704v-contig00010' as the query identifier Example 2: ... 2 gi|9625521|ref|NP_039778.1| putative integrase [Sulfolobus virus 1] >gi|138570|sp|P20214.1|INTG_SSV1 RecName: Full=Probable integrase >gi|46705|emb|CAA30211.1| ORF D-335 [Sulfolobus spindle-shaped virus 1] NP_039778 335 ... The following syntax, --hit-id-getter "{'id': , 'definition': , 'accession': }" together with no provided hit collection, will create custom objects as hits which will be a dictionary with the three keys 'id', 'definition' and 'accession'
